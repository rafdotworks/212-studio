# Changelog

All notable changes to 212 Studio website.

## [1.0.0] - 2026-02-02

### Initial Launch

**Rebrand from Studio Värn to 212 Studio**
- Updated all brand references across the site
- Changed title and meta descriptions
- Updated package name to `studio212`

**Content & Structure**
- Simplified to 3 core pages: Index, Information, Contact
- Removed: Selected Works, Atelier, Publications pages
- Featured manifesto prominently on Information page
- Added studio founders: Marianne Ventre and Raf V.

**Design System**
- Established minimal, editorial design language
- Warm neutral palette (#F4F3F0 background)
- Helvetica Neue typography
- Grid-based layout with sticky sidebar
- Vertical project stack on homepage

**Projects Display**
- Implemented vertical scroll format for projects
- Each project includes: hero image, description, detail images
- 3 placeholder projects with abstract visualizations
- Generous 100px spacing between projects
- Consistent metadata display (category + year)

**Navigation & Layout**
- Simplified navigation: Index, Information, Contact
- Sticky navigation at viewport top
- Metadata block fixed at viewport bottom
- 240px sidebar with responsive breakpoint at 768px

**Contact**
- Removed contact form for simplicity
- Direct email link: raf@212studio.works
- Clean, minimal contact section

**Location & Timing**
- Updated to 2026
- Location: Lisbon — London
- Status: Independent
- Updated date stamp: February 2026

**Technical**
- React 18 with Vite 5
- React Router 6 for routing
- Inline styles for simplicity
- pnpm package manager

**Deployment**
- Initial deployment to Vercel
- Connected GitHub repository for auto-deploy
- Production URL: https://212-studio-pnckpgmqx-raf-project.vercel.app
- GitHub: https://github.com/rafdotworks/212-studio

**Documentation**
- Comprehensive README with setup and usage
- Project philosophy and design decisions
- Working with projects guide
- Deployment workflow

### Design Philosophy

The site embodies 212 Studio's manifesto principles:
- Intent before execution
- Simplicity over noise
- Coherence over novelty
- Restraint in what is included
- Design that holds up to use
